# Epic Universe Day Planner

Local web app to plan a day at Universal Epic Universe Orlando.
Built for **May 25, 2026 (Memorial Day Monday)** by default; date is configurable.

## What it does

- **Wait-time predictor**: a scikit-learn GradientBoostingRegressor predicts
  how long the line will be for every ride at every hour on any future date.
  Trained on a mix of historical queue-times.com data and synthetic samples
  generated from a hand-calibrated crowd-factor model (holidays, day-of-week,
  month seasonality, Epic's novelty decay).
- **Drag-and-drop itinerary**: dark-themed timeline with hour rows. Drag
  attractions from the left sidebar onto the day. Move bars to reposition.
- **AI optimizer**: priority-weighted greedy scheduler with show-time anchoring.
  Tells you the lowest-wait order, flags infeasibility, lists warnings.
- **Anthropic API integration** (BYO key): "Ask Claude" floating chat with
  your itinerary as context, plus a one-click "critique my plan" feature.

## Run it

### Backend

```sh
python3 -m venv .venv
.venv/bin/pip install -r backend/requirements.txt
.venv/bin/python scripts/train_model.py        # one-time, ~15s
.venv/bin/uvicorn backend.main:app --reload    # http://localhost:8000
```

API docs at http://localhost:8000/docs.

### Frontend

```sh
cd frontend
npm install
npm run dev        # http://localhost:5173
```

The Vite dev server proxies `/api` → `localhost:8000`, so just open
`http://localhost:5173`.

### Optional: collect real wait-time data

```sh
# Snapshot current queue-times.com waits into data/live_log.csv
.venv/bin/python scripts/fetch_historical.py
```

Run on a cron every 15-30 min to build a real historical dataset. Re-running
`train_model.py` will pick up the live log and oversample it 5× during training.

## Project layout

```
backend/                FastAPI + ML
  data/attractions_db.py    # ground-truth attraction catalog (verified May 2026)
  data/queue_times_client.py
  ml/crowd_factors.py       # day-type + hourly + novelty multipliers
  ml/features.py            # model feature engineering
  ml/model.py               # WaitTimePredictor (loads saved_model.pkl)
  ml/saved_model.pkl        # generated by scripts/train_model.py
  routers/                  # FastAPI endpoints
frontend/                  React + Vite + TypeScript + Tailwind + dnd-kit
scripts/                  one-off Python scripts
```

## AI features (optional)

Add your Anthropic API key via the "Add AI key" button in the top bar. It's
stored in `localStorage` and sent per-request as an `X-Anthropic-Key` header
to your local backend, which forwards to the Anthropic API. The server never
persists it.
